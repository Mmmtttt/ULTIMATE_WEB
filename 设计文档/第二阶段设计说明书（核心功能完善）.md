## 1. 架构演进设计

### 1.1 第一阶段架构回顾
```
第一阶段：简单三层架构
Routes → Utils → Data

问题：
1. 业务逻辑分散在Routes中
2. 代码复用性低
3. 单元测试困难
```

### 1.2 第二阶段架构优化
```
优化后：四层架构
┌─────────────────────────────────────┐
│            Routes (路由层)           │
│    职责：请求接收、参数校验、响应封装    │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│          Services (业务层)           │
│    职责：业务逻辑、数据校验、流程编排    │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│           Utils (工具层)             │
│    职责：数据访问、文件操作、通用工具    │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│            Data (数据层)             │
│    职责：JSON文件、图片文件存储         │
└─────────────────────────────────────┘
```

### 1.3 架构优化收益
| 优化点 | 收益 |
|--------|------|
| 引入Services层 | 业务逻辑集中管理，便于维护和测试 |
| 职责分离 | 每层职责单一，降低耦合度 |

#### 2.1.6 搜索算法设计
```
多维度搜索策略：
1. 漫画ID精准匹配（优先级最高）
   - 输入纯数字或符合ID格式
   - 直接匹配comic_id字段

2. 漫画名称模糊匹配（优先级次之）
   - 转换为小写
   - 包含匹配（keyword in title）

3. 标签精准匹配（优先级最低）
   - 匹配tag名称
   - 返回绑定该标签的所有漫画

实现优化：
- 缓存标签映射关系（tag_name → tag_id）
- 避免重复读取JSON文件
```

#### 2.1.7 标签筛选算法设计
```
筛选逻辑：
1. 包含标签筛选（AND逻辑）
   - 返回同时绑定所有包含标签的漫画
   - 实现：取多个标签漫画列表的交集

2. 排除标签筛选（NOT逻辑）
   - 返回未绑定任意排除标签的漫画
   - 实现：过滤掉绑定排除标签的漫画

3. 组合筛选
   - 先执行包含筛选
   - 再执行排除筛选
   - 返回最终结果

算法复杂度优化：
- 小集合作为基准，减少比较次数
- 使用集合（Set）操作提高效率
```

#### 2.3.2 评分验证
```python
def validate_score(score):
    """
    评分验证规则：
    1. 必须是数字
    2. 范围1-12
    3. 精度0.5（score * 2 必须是整数）
    """
    if not isinstance(score, (int, float)):
        return False, "评分必须是数字"
    
    if score < 1 or score > 12:
        return False, "评分范围1-12"
    
    if (score * 2) % 1 != 0:
        return False, "评分精度0.5"
    
    return True, "验证通过"
```

```
排序支持：
- 评分从高到低
- 评分从低到高
```

```javascript
// composables/usePageTurn.js
export function usePageTurn(totalPage) {
  const currentPage = ref(1);
  const isAnimating = ref(false);
  const preloadedPages = ref(new Set());
  
  // 预加载图片
  const preloadImage = (pageNum) => {
    if (preloadedPages.value.has(pageNum)) return;
    
    const img = new Image();
    img.src = getImageUrl(pageNum);
    preloadedPages.value.add(pageNum);
  };
  
  // 预加载范围
  const preloadRange = () => {
    const range = 3;
    for (let i = -range; i <= range; i++) {
      const page = currentPage.value + i;
      if (page >= 1 && page <= totalPage) {
        preloadImage(page);
      }
    }
  };
  
  // 下一页
  const nextPage = () => {
    if (isAnimating.value || currentPage.value >= totalPage) return;
    
    isAnimating.value = true;
    currentPage.value++;
    preloadRange();
    
    setTimeout(() => {
      isAnimating.value = false;
    }, 200);
  };
  
  // 上一页
  const prevPage = () => {
    if (isAnimating.value || currentPage.value <= 1) return;
    
    isAnimating.value = true;
    currentPage.value--;
    preloadRange();
    
    setTimeout(() => {
      isAnimating.value = false;
    }, 200);
  };
  
  return {
    currentPage,
    nextPage,
    prevPage,
    preloadRange
  };
}
```

```vue
<template>
  <div class="virtual-list" ref="container" @scroll="onScroll">

    </div>

<script setup>
const props = defineProps({
  items: Array,
  itemHeight: Number
});

const container = ref(null);
const visibleCount = ref(10);
const startIndex = ref(0);

const visibleItems = computed(() => {
  return props.items.slice(startIndex.value, startIndex.value + visibleCount.value);
});

const totalHeight = computed(() => props.items.length * props.itemHeight);

const onScroll = () => {
  const scrollTop = container.value.scrollTop;
  startIndex.value = Math.floor(scrollTop / props.itemHeight);
};
</script>
```

## 8. 风险规避

### 8.1 数据一致性风险
| 风险 | 规避措施 |
|------|----------|
| 标签删除后漫画仍引用 | 删除标签时遍历所有漫画清理引用 |
| 并发修改JSON | 文件锁保证原子性写入 |
| 部分更新失败 | 事务机制，失败时回滚 |

### 8.2 兼容性风险
| 风险 | 规避措施 |
|------|----------|
| 浏览器不支持新特性 | 使用polyfill，渐进增强 |
| 触屏手势冲突 | 合理设置touch-action |
| 低性能设备卡顿 | 虚拟列表，减少DOM节点 |

### 8.3 安全风险
| 风险 | 规避措施 |
|------|----------|
| 路径遍历攻击 | 严格校验comic_id格式 |
| JSON注入 | 输入校验，转义特殊字符 |
| 文件上传漏洞 | 本项目不涉及文件上传 |
