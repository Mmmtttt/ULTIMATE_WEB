## 修复内容

### Bug 1: 详情页清单保存按钮无响应
**问题描述**：
- 点击清单弹窗中的保存按钮没有反应
- 前后端都没有日志输出

**问题分析**：
1. `customLists` 使用了 `listStore.customLists`，该计算属性过滤掉了收藏清单
2. 弹窗打开时可能清单列表未加载完成
3. `selectedListIds` 初始化可能有问题

**修复方案**：
1. 修改 `customLists` 计算属性，使用 `listStore.lists` 显示所有清单
2. 在弹窗打开时检查并加载清单列表
3. 使用数组解构 `[...]` 初始化 `selectedListIds`，避免引用问题
4. 添加调试日志便于排查

**涉及文件**：
- `comic_frontend/src/views/ComicDetail.vue`

**核心代码**：
```javascript
const customLists = computed(() => listStore.lists || [])

watch(showListPopup, async (val) => {
  if (val) {
    if (listStore.lists.length === 0) {
      await listStore.fetchLists()
    }
    if (comic.value) {
      selectedListIds.value = [...(comic.value.list_ids || [])]
    }
  }
})
```

### Bug 2: 标签管理批量操作不显示漫画
**问题描述**：
- 标签管理页面的批量操作Tab中，漫画列表为空
- 前后端都没有日志输出

**问题分析**：
`fetchComicList` 函数中错误地使用了 `response.data`，但 `comicStore.fetchComics()` 直接返回数组，不是 `{ data: [] }` 格式

**修复方案**：
修改 `fetchComicList` 函数，直接使用返回的数组

**涉及文件**：
- `comic_frontend/src/views/TagManage.vue`

**核心代码**：
```javascript
const fetchComicList = async () => {
  try {
    const response = await comicStore.fetchComics()
    if (response) {
      comicList.value = response  // 直接使用返回的数组
    }
  } catch (error) {
    console.error('获取漫画列表失败:', error)
  }
}
```
