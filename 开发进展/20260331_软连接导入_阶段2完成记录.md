# 软连接导入 阶段2完成记录（2026-03-31）

## 本阶段目标
- 开启 `softlink_ref` 服务端路径导入，不移动、不删除源文件。
- 在导入会话阶段构建“虚拟目录树”，支持目录 + 压缩包混合结构。
- 导出/提交流程复用现有层级标记逻辑，软连接模式写入 `soft_ref` 元数据。
- 通过集成测试验证阶段目标，不引入 E2E 门禁。

## 已实现
- 后端 `create_session_from_path` 新增 `archive_password`（可选），`softlink_ref` 模式正式可用。
- 新增虚拟树扫描能力：
  - 文件夹递归扫描（不复制、不解压到源目录）。
  - 压缩包虚拟展开（zip/rar/7z 按环境能力处理）。
  - 嵌套压缩包深度解析（有大小与深度保护）。
- 新增软连接定位符 `softref://...`，用于标识压缩包内部目录路径。
- `export_session_items` / `commit_session_import` 改为“按会话模式分流”：
  - `copy_safe/move_huge` 维持原逻辑。
  - `softlink_ref` 走虚拟树导出，并写入：
    - `storage_mode = "soft_ref"`
    - `soft_ref_locator`
    - `import_source`
  - 软连接模式提交不做数据复制/移动。
- 前端路径导入新增可选“压缩包密码”输入（仅软连接模式显示），通过 API 透传。
