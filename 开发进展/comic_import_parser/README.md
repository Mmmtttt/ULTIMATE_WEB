# 本地漫画导入解析器

这是一个可以直接运行的示例项目，用来解决下面这类本地漫画导入问题：

- 导入源可能是文件夹，也可能是 `.zip / .rar / .7z`
- 任意层级都可能继续嵌套压缩包或文件夹
- 后端最终只想拿到“干净目录树”（目录树中不再包含压缩包，只剩文件夹和图片）
- 目录树中“作者层”和“作品层”无法稳定自动识别，需要用户在前端手动指定
- 用户一旦把某个目录标记为“作者层”或“作品层”，则它的**同级目录**会被一起赋予同样语义

最终导出的 JSON 结构如下：

```json
[
  {
    "作者名称": "作者1",
    "作品名称": "作品1",
    "作品文件地址": "/absolute/path/to/clean/作者1/2020/作品1"
  }
]
```

## 功能概览

1. **递归解包**：把任意层级的 `.zip/.rar/.7z` 全部解到临时缓存区
2. **清洗目录**：生成一个不再包含压缩包的“干净目录树”
3. **前端可视化**：目录树展示每个文件夹的图片数量
4. **手动标记层级**：
   - 选择某个目录
   - 点击“把这一层标为作者层”或“把这一层标为作品层”
   - 该目录所在层的全部同级目录会一起生效
5. **生成 JSON**：输出作者名称、作品名称、作品目录路径
6. **下载 JSON**：把解析结果落成 `result.json`

## 启动方式

```bash
cd comic_import_parser
python -m venv .venv
source .venv/bin/activate        # Windows 用 .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app:app --reload --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

## 两种导入方式

### 1）按本机路径导入

适合你的本地站点前后端就在同一台机器上运行。

直接在页面里输入：

- Windows 示例：`D:\漫画\作者合集.zip`
- macOS/Linux 示例：`/Users/me/comics/作者合集`

### 2）浏览器上传导入

- 可以上传一个或多个压缩包
- 可以直接上传整个文件夹
- 文件夹结构会在后端被还原，再做递归清洗

## 交互逻辑

### 标记“作者层”

当你选择某个目录并点击“把这一层标为作者层”时：

- 实际上标记的是这个目录的**父目录下面的整层子目录**
- 也就是这个目录的所有同级目录，都会被一起视为作者目录

### 标记“作品层”

逻辑同上：

- 你点中某个目录后
- 点击“把这一层标为作品层”
- 它的所有同级目录都会一起视为作品目录

### 作者名如何解析

对每个作品目录，程序会向上回溯最近的一个“作者层”祖先目录：

- 如果找到了，就把那个目录名作为作者名称
- 如果没找到，作者名称输出为空字符串

### 作品如何去重

如果你误把父层和子层都标成“作品层”，程序只会导出**最深层**的作品目录，避免重复。

## 临时缓存区说明

每次导入都会在 `workspace/<session_id>/` 下生成独立会话目录：

```text
workspace/<session_id>/
├─ input/     # 原始导入内容
├─ clean/     # 清洗后的干净目录树
├─ scratch/   # 解压时的临时目录
├─ tree.json  # 当前目录树快照
├─ meta.json  # 会话元数据
└─ result.json
```

## 关于 .rar 支持

`.zip` 使用 Python 标准库即可处理。

`.7z` 依赖 `py7zr`。

`.rar` 依赖 `rarfile`，它通常还需要系统中存在一个可用的解压后端（例如 `unrar`、`unar`、`7zip` 或 `bsdtar`）。如果你的环境里没有这些后端，`.rar` 文件会在页面“导入警告”里显示跳过原因。
