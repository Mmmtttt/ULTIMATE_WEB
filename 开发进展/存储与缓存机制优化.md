# 存储与缓存机制优化报告

**日期**: 2026-03-03

## 一、本次优化内容

### 1. 三级备份策略优化

**问题**: 原有备份策略每个层级只保留1个文件，频繁覆盖

**解决方案**:
- 每个层级保留3个版本备份
- 备份文件名带时间戳: `{base_name}_tier1_20260303_153000.bkp`
- 自动轮转：超过最大数量时删除最旧的

**备份策略**:
| 层级 | 频率 | 保留数量 | 说明 |
|------|------|----------|------|
| Tier 1 | 每10分钟 | 3个 | 30分钟内的历史 |
| Tier 2 | 每小时 | 3个 | 3小时内的历史 |
| Tier 3 | 每天 | 3个 | 3天内的历史 |

**相关文件**: `infrastructure/backup_manager.py`

---

### 2. 永久删除漫画时同步清理文件

**问题**: 永久删除漫画只从JSON移除，图片和封面文件残留

**解决方案**:
- 永久删除时同时删除:
  - 漫画正文图片目录: `data/pictures/JM/<漫画ID>/`
  - 漫画封面文件: `static/cover/JM/<漫画ID>.jpg`
  - 推荐页缓存目录: `data/recommendation_cache/JM/<漫画ID>/`

**相关文件**:
- `application/comic_app_service.py` - 主页漫画删除
- `application/recommendation_app_service.py` - 推荐页漫画删除

---

### 3. 日志文件机制

**结论**: 日志已有轮转机制，无需优化

**现有机制**:
- 单文件最大100MB
- 保留20个备份
- 使用 `RotatingFileHandler`

---

### 4. 缓存设置API

**新增配置项**:
```json
{
  "cache_config": {
    "recommendation_cache_max_size_mb": 5120,
    "cache_ttl_seconds": 3600
  }
}
```

**新增API**:
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/config` | GET | 获取配置（含缓存统计） |
| `/api/v1/config` | PUT | 更新配置（含缓存容量） |
| `/api/v1/config/cache/stats` | GET | 获取缓存统计 |
| `/api/v1/config/cache/clear` | DELETE | 清除所有缓存 |
| `/api/v1/config/cache/orphan` | DELETE | 清理孤立缓存 |

**相关文件**:
- `domain/config/entity.py` - 添加CacheConfig
- `api/v1/config.py` - 添加缓存管理API

## 二、删除缓存说明

点击"删除缓存"时，会删除以下内容:

1. **推荐页漫画缓存** - `data/recommendation_cache/`
   - JM平台: `data/recommendation_cache/JM/<漫画ID>/`
   - PK平台: `data/recommendation_cache/PK/<漫画ID>/`
   - 共用索引: `data/meta_data/recommendation_cache_index.json`

2. **清理孤立缓存**
   - 删除索引中没有记录但磁盘存在的缓存目录

**不会删除**:
- 漫画正文图片 (`data/pictures/`)
- 漫画封面 (`static/cover/`)
- 用户配置 (`data/meta_data/user_config.json`)
- 数据库文件
