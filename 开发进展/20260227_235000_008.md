# 20260227_235000_008 - 阅读页面滑动和切换按钮修复

## 问题描述

1. **滑动功能失效**：修改后只能点击下一页切换，无法左右滑动
2. **切换按钮不显示**：没有看到上下/左右切换的按钮

## 问题原因

### 1. 滑动功能失效
- 之前移除了 `@scroll` 事件监听，导致无法检测滑动
- `scroll-snap` 配置不当，影响滑动流畅度

### 2. 切换按钮不显示
- 使用了 `van-icon` 组件，但 Vant 图标需要单独导入或配置
- 图标名称可能不正确

## 修复方案

### 1. 修复滑动功能
- 添加 `@scroll` 事件监听，检测滚动位置
- 使用 `scroll-snap-type` 和 `scroll-snap-align` 实现流畅的滑动吸附
- 添加 `-webkit-overflow-scrolling: touch` 优化移动端滑动

### 2. 修复切换按钮
- 将 `van-icon` 替换为普通文本按钮
- 使用简单的 `<span>` 元素显示当前模式

```vue
<!-- 左右模式 -->
<div 
  v-if="pageMode === 'left_right'" 
  class="left-right-mode" 
  ref="leftRightContainer"
  @scroll="handleHorizontalScroll"

<!-- 上下模式 -->
<div 
  v-else 
  class="up-down-mode" 
  ref="upDownContainer" 
  @scroll="handleVerticalScroll"

```

```javascript
// 处理左右滚动
const handleHorizontalScroll = () => {
  if (pageMode.value === 'left_right' && leftRightContainer.value) {
    const scrollLeft = leftRightContainer.value.scrollLeft
    const pageWidth = window.innerWidth
    const newPage = Math.round(scrollLeft / pageWidth) + 1
    if (newPage !== currentPage.value && newPage >= 1 && newPage <= totalPage.value) {
      currentPage.value = newPage
      saveProgress()
    }
  }
}

// 处理上下滚动
const handleVerticalScroll = () => {
  if (pageMode.value === 'up_down' && upDownContainer.value) {
    const scrollTop = upDownContainer.value.scrollTop
    const pageHeight = window.innerHeight - 46
    const newPage = Math.round(scrollTop / pageHeight) + 1
    if (newPage !== currentPage.value && newPage >= 1 && newPage <= totalPage.value) {
      currentPage.value = newPage
      saveProgress()
    }
  }
}
```

## 技术说明

### Scroll Snap 技术
- `scroll-snap-type: x mandatory` - 强制水平吸附
- `scroll-snap-type: y mandatory` - 强制垂直吸附
- `scroll-snap-align: start` - 吸附到元素起始位置
- `scroll-snap-stop: always` - 总是停止在吸附点

### 移动端优化
- `-webkit-overflow-scrolling: touch` - 启用惯性滚动
- `scroll-behavior: smooth` - 平滑滚动动画

### 页码同步
- 滚动时实时计算当前页码
- 页码变化时自动保存阅读进度
- 切换模式时保持当前页码不变
