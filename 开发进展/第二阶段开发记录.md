## 四、第三阶段开发展望

### 4.1 后端开发计划

#### 清单管理模块（核心功能）

| 接口 | 请求方式 | 功能 | 优先级 |
|------|---------|------|--------|
| `/api/v1/list/add` | POST | 新增清单 | 🔴 高 |
| `/api/v1/list/list` | GET | 获取清单列表 | 🔴 高 |
| `/api/v1/list/detail` | GET | 获取清单详情 | 🔴 高 |
| `/api/v1/list/edit` | PUT | 编辑清单 | 🟡 中 |
| `/api/v1/list/delete` | DELETE | 删除清单 | 🔴 高 |
| `/api/v1/list/comic/bind` | PUT | 漫画加入清单 | 🔴 高 |
| `/api/v1/list/comic/remove` | DELETE | 漫画移出清单 | 🔴 高 |

**技术要点：**
- 清单 ID 生成规则：`list_{timestamp}`
- 默认清单 ID：`list_favorites`（不可删除）
- 清单删除时清理所有漫画的 `list_ids` 引用
- 批量操作支持去重和部分失败回滚

#### 配置管理模块（核心功能）

| 接口 | 请求方式 | 功能 | 优先级 |
|------|---------|------|--------|
| `/api/v1/config` | GET | 获取配置 | 🔴 高 |
| `/api/v1/config` | PUT | 更新配置 | 🔴 高 |

**配置项：**
```json
{
  "user_config": {
    "preload_num": 3,
    "default_page_mode": "left_right",
    "default_background": "white",
    "auto_hide_toolbar": true,
    "show_page_number": true
  }
}
```

**技术要点：**
- 使用单例模式 `ConfigManager`
- 配置校验规则（类型、范围、选项）
- 配置变更事件通知
- 默认配置管理

#### 工具类优化

| 工具类 | 优化内容 | 优先级 |
|--------|---------|--------|
| `BatchProcessor` | 批量处理器，支持分批处理、重试 | 🟡 中 |
| `ListService` | 清单业务逻辑，包含 CRUD 操作 | 🔴 高 |
| `ConfigManager` | 配置管理器，单例模式 | 🔴 高 |
