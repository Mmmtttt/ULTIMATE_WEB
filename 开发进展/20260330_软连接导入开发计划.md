# 软连接导入开发计划（2026-03-30）

## 1. 目标与边界
- 目标：在“服务端路径导入”中新增两种可选模式。
- 模式A：`硬链接导入（移动源文件）`，行为等同当前 `move_huge`（即移动导入）。
- 模式B：`软连接导入（不移动源文件）`，不改动源文件，不解压源压缩包，阅读时可无感访问目录/压缩包内容。
- 仅改服务端路径导入链路；上传导入链路暂不改。
- 当前先以漫画实现，但架构必须为后续视频导入预留接口。

## 2. 已确认需求决策
- 快照策略：以导入时为准，后续源目录变化不追踪；读取失败时报错“源文件不存在/不可读”。
- 密码存储：明文保存（后续可升级为加密存储）。
- 同步策略：若软连接资源跨设备无法完整传输，则跳过（必要时连元数据也跳过）。
- 文案：可使用“硬链接导入（移动源文件）”。
- 测试策略：以集成测试为主，不以 E2E 为门禁。

## 3. 总体技术方案（分层）
- 导入层：新增 `import_mode` 枚举与模式分流，不破坏现有 `copy_safe/move_huge`。
- 索引层：为软连接导入新增“虚拟资源索引（Virtual Source Index）”，保存目录树、页定位、压缩包层级信息。
- 读取层：新增统一读取适配接口，使阅读链路不感知本地目录与压缩包来源差异。
- 密码层：新增密码仓库（archive fingerprint -> password），首次读取失败时请求用户输入并缓存。
- 同步层：对软连接资源做显式跳过策略，避免同步不可用状态污染对端。

## 4. 分阶段实施（按顺序）

## 阶段1（低风险，低复杂度）
- 难度：低
- 目标：
  - 路径导入UI从“超大文件移动导入开关”改为“导入模式选择（硬链接/软连接）”。
  - 后端支持 `hardlink_move` 别名并映射到现有 `move_huge`。
  - 预留 `softlink_ref` 模式入口，当前明确返回“暂未启用”（禁止静默降级）。
  - 不改变既有 `copy_safe/move_huge` 真实行为。
- 验收：
  - 选择“硬链接导入（移动源文件）”时，与原 `move_huge` 行为一致。
  - 选择“软连接导入”时，服务端返回明确错误码/错误消息。
- 集成测试：
  - `hardlink_move` 别名行为测试（路径导入+提交后源目录被移动）。
  - `softlink_ref` 暂未启用返回 400 测试。

## 阶段2（中风险，高复杂度）
- 难度：高
- 目标：
  - 软连接导入解析器：不解压落地，直接扫描“目录+压缩包+嵌套压缩包”形成虚拟目录树。
  - 生成虚拟作品定位信息（work locator），提交时入库为 `storage_mode=soft_ref`。
  - 为漫画保存外部索引文件（避免主库膨胀）。
- 验收：
  - 不修改源目录/源压缩包。
  - 可在导入标记页看到压缩包内部目录结构。
  - 可成功提交并落库。
- 集成测试：
  - 普通目录导入（软连接模式）。
  - 压缩包导入（zip/7z/rar，按环境能力分层）。
  - 多层嵌套压缩包目录树构建测试。

## 阶段3（中风险，高复杂度）
- 难度：高
- 目标：
  - 阅读链路适配：`/comic/images`、`/comic/image`、封面生成、页数统计对 `soft_ref` 无感。
  - 首次访问加密压缩包时返回“需要密码”；提交密码后可继续读取。
  - 密码仓库明文落盘，按 archive fingerprint 复用。
- 验收：
  - 阅读页无需前端改造即可翻页。
  - 压缩包密码仅首次请求，后续自动读取。
  - 源文件缺失时返回明确错误。
- 集成测试：
  - 加密 zip/7z/rar 首次失败 + 提交密码 + 再次成功。
  - 嵌套压缩包读取首图、分页、封面生成。
  - 源路径删除后读取失败报错。

## 阶段4（中风险，中复杂度）
- 难度：中
- 目标：
  - 删除/清理安全：软连接导入删除漫画时仅删索引与数据库，不删源文件。
  - 会话恢复：软连接模式中断后可恢复提交。
  - 同步跳过策略：软连接资源默认跳过同步（含元数据策略按最终规则实现）。
- 验收：
  - 永久删除软连接漫画不影响源目录。
  - 恢复会话可继续完成。
  - 同步日志明确标识跳过原因。
- 集成测试：
  - 删除安全测试。
  - 恢复导入测试。
  - 同步跳过测试。

## 阶段5（低风险，中复杂度）
- 难度：中
- 目标：
  - 重构抽象：将 `soft_ref` 读取协议抽象为可扩展接口，为视频导入复用预留。
  - 文档完善：开发者文档新增“软连接导入架构与限制”。
- 验收：
  - 代码接口抽象清晰（comic/video 可共用）。
  - 文档可指导二次开发。
- 集成测试：
  - 回归测试全集通过。

## 5. 风险清单与规避
- RAR/7z 平台依赖差异：在阶段2能力探测，缺失时明确提示，不静默失败。
- 大型固实压缩包随机读慢：阶段3增加读取缓存与超时保护。
- 嵌套层级过深：限制最大递归层级，防止资源耗尽。
- 源路径权限变化：读取时统一错误码并提示用户。

## 6. 测试门禁策略
- 主门禁：集成测试（后端API + 文件系统 + 压缩包读取）。
- 不作为门禁：E2E（本需求核心在后端读取与索引，不依赖复杂前端交互）。
- 每阶段必须满足：
  - 新增集成测试全部通过。
  - 既有本地导入相关集成测试无回归。
  - 前端构建通过。
