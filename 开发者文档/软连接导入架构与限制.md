# 软连接导入架构与限制

## 1. 目标
- 在不复制/移动源文件的前提下，支持“服务端路径导入”的本地漫画读取。
- 阅读层对来源无感：目录源、压缩包源统一通过读取协议输出分页图片流。
- 保持现有功能兼容，并为后续“本地视频导入”预留可复用接口。

## 2. 核心结构
- 导入侧：`local_comic_import_service` 在 `softlink_ref` 模式写入：
  - `storage_mode = "soft_ref"`
  - `soft_ref_locator`（软连接定位符）
  - `import_source`（来源路径）
- 读取侧：`softref_comic_reader` 负责：
  - 定位符解析
  - 分页索引构建与缓存
  - 目录/压缩包图片读取
  - 密码仓库读写
- 协议层：`application/softref_reader_protocol.py`
  - `SoftRefReaderProtocol`：统一读取协议
  - `softref_reader_registry`：按内容类型注册 reader（当前已注册 `comic`）

## 3. 协议定义（面向扩展）
- `is_soft_ref_content(content_id)`: 判断是否为 soft_ref 资源。
- `get_page_count(content_id)`: 返回页数（或帧数）。
- `get_image_stream(content_id, page_num)`: 返回二进制流与 mime。
- `set_archive_password(content_id, archive_fingerprint, password)`: 保存密码。
- `get_password_required_payload(content_id, exc)`: 返回前端可消费的挑战对象。

当前漫画 API 和封面回填逻辑已通过 `require_softref_reader("comic")` 获取实现，不再依赖具体类名，后续可并行接入 `video`。

## 4. 数据与缓存
- 密码仓库：`data/cache/comic_softref_passwords.json`（当前为明文存储）。
- 嵌套压缩包缓存：进程内 LRU（大小受环境变量控制）：
  - `SOFTREF_NESTED_ARCHIVE_CACHE_MAX_BYTES`
  - `SOFTREF_NESTED_ARCHIVE_CACHE_SINGLE_MAX_BYTES`

## 5. 当前限制
- 软连接导入模式下，前端约束为“单层压缩包”导入体验；更深层嵌套不作为默认导入路径能力承诺。
- 源路径以导入时快照为准：后续源文件被移动/删除后，阅读会返回“源文件不存在或不可访问”。
- 密码目前明文保存，适用于单机可信环境。
- 同步链路对 soft_ref 的跨设备可达性天然受限（目标设备通常无同路径源文件）。

## 6. 视频扩展建议
- 新增 `video` reader 并注册：
  - `register_softref_reader("video", video_softref_reader)`
- 视频侧沿用同一协议，替换：
  - 分页语义 -> 时间片/关键帧语义
  - `get_image_stream` -> `get_media_stream`（若后续需要，可在协议层做 v2 扩展）
- 建议先保持协议向后兼容，再引入 v2 协议避免破坏漫画链路。
