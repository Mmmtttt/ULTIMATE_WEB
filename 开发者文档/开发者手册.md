### 1.4 环境安装

#### Windows 环境（推荐 Conda）

```powershell
# 1. 克隆仓库（包含子模块）
git clone --recursive https://github.com/Mmmtttt/ULTIMATE_WEB.git
cd ULTIMATE_WEB

# 2. 一键安装构建环境（需要 Conda）
conda activate base
powershell -ExecutionPolicy Bypass -File .\scripts\install_build_env_windows_base.ps1

# 3. 重启终端后验证
conda activate base
python --version
node --version
npm --version
java --version
sdkmanager.bat --version
adb --version
```

#### Linux/macOS 环境

```bash
# 1. 克隆仓库
git clone --recursive https://github.com/Mmmtttt/ULTIMATE_WEB.git
cd ULTIMATE_WEB

# 2. 安装后端依赖
python3 -m pip install --upgrade pip
python3 -m pip install -r comic_backend/requirements.txt

# 3. 安装前端依赖
cd comic_frontend
npm ci
```

### 1.5 开发环境启动

#### Windows

```powershell
# 一键启动前后端
.\scripts\start_project.ps1

# 或分开启动
.\scripts\start_backend.ps1   # 后端 http://localhost:5000
.\scripts\start_frontend.ps1  # 前端 http://localhost:5173
```

#### Linux/macOS

```bash
# 后端
./scripts/start_backend.sh

# 前端
./scripts/start_frontend.sh
```

## 2. 打包与发布

### 2.1 打包方式概览

| 方式 | 适用场景 | 产物 |
|------|----------|------|
| GitHub Actions（推荐） | 正式发布 | Windows EXE + Linux 可执行文件 + Android APK |
| 本地打包 Windows | 快速验证 | Windows EXE |
| 本地打包 Android | Android 测试 | Android APK |
| 本地打包 Linux | Linux 测试 | Linux 可执行文件 |

### 2.2 GitHub Actions 自动打包（推荐）

#### 方式一：推送 Tag 触发

```bash
# 创建并推送版本标签
git tag v1.0.0
git push origin v1.0.0
```

#### 方式二：手动触发

1. 打开 GitHub 仓库页面
2. 点击 **Actions** 标签
3. 选择 **release-three-platforms** 工作流
4. 点击 **Run workflow**

### 2.3 本地打包

#### Windows 本地打包

```powershell
# 激活环境
conda activate base

# 一条命令完成构建+打包
python .\scripts\release_unified.py --targets "windows" --execute --build-output output\local_stage --package-output output\local_packages
```

#### Windows + Android 同时打包

```powershell
python .\scripts\release_unified.py --targets "windows,android" --execute --build-output output\multi_target --package-output output\packages
```

#### Linux 本地打包

```bash
python3 scripts/release_unified.py --targets linux --execute --build-output output/local_stage --package-output output/local_packages
```

### 2.4 打包脚本说明

| 脚本 | 用途 |
|------|------|
| `scripts/release_unified.py` | 统一发布入口，串联构建与打包 |
| `scripts/build_unified.py` | 统一构建阶段，产出 staging 工作区 |
| `scripts/package_unified.py` | 统一打包阶段，生成平台包 |
| `scripts/verify_packaging_summary.py` | 校验打包结果一致性 |

### 3.3 运行测试

#### 一键运行全部测试

```powershell
# Windows
.\tests\run_test_gate.ps1

# Linux/macOS
bash ./tests/run_test_gate.sh
```

#### 分开运行

```bash
# Integration 测试
python -m pytest tests/features -m integration

# E2E 测试
python tests/tools/run_e2e.py
```

#### 运行特定测试

```bash
# 运行特定功能的 Integration 测试
python -m pytest tests/features/library_browse/integration -v

# 运行特定 E2E 测试
npx playwright test tests/features/library_browse/e2e/library_open_detail.spec.js
```

### 3.4 测试数据隔离

测试使用独立沙箱环境，不影响真实数据：

```
tests/.runtime/
├── e2e/                    # E2E 测试沙箱
│   ├── data/              # 数据库文件
│   ├── static/            # 静态资源
│   └── config/            # 配置文件
└── integration/            # Integration 测试沙箱
```

**环境变量控制：**

```bash
SERVER_CONFIG_PATH=tests/.runtime/integration/config/server_config.json
THIRD_PARTY_CONFIG_PATH=tests/.runtime/integration/config/third_party_config.json
BACKEND_ENABLE_THIRD_PARTY=0
```

### 3.5 测试用例编写规范

每个测试用例必须包含：

```python
def test_feature_name():
    """
    用例描述：
    
    1. 用例目的：
       验证 XXX 功能在 YYY 条件下的行为
    
    2. 测试步骤：
       - 步骤1：准备数据
       - 步骤2：执行操作
       - 步骤3：验证结果
    
    3. 预期结果：
       - 结果1：返回状态码 200
       - 结果2：数据正确保存
    
    4. 历史变更：
       - 2026-03-25：初始创建
    """
    pass
```

### 3.6 CI 测试门禁

GitHub Actions 自动执行测试门禁：

```yaml
# .github/workflows/test-gate.yml
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Run Integration Tests
        run: python -m pytest tests/features -m integration
      
      - name: Run E2E Tests
        run: python tests/tools/run_e2e.py
```

### 5.2 分层职责

| 层级 | 目录 | 职责 |
|------|------|------|
| API 层 | `api/v1/` | 接收 HTTP 请求，参数校验，调用应用服务，返回响应 |
| 应用服务层 | `application/` | 业务逻辑编排，事务管理，跨领域协调 |
| 领域层 | `domain/` | 实体定义，业务规则，仓储接口 |
| 基础设施层 | `infrastructure/` | 数据持久化，外部服务集成，技术实现 |
| 第三方适配 | `third_party/` | 第三方平台 API 封装，数据格式转换 |

### 6.3 适配器接口规范

所有平台适配器继承 `BaseAdapter`，必须实现以下方法：

```python
class BaseAdapter:
    """适配器基类"""
    
    def get_platform(self) -> Platform:
        """返回平台类型"""
        raise NotImplementedError
    
    def search_comics(self, keyword: str, max_pages: int = 1) -> List[Dict]:
        """搜索内容"""
        raise NotImplementedError
    
    def get_comic_detail(self, album_id: str) -> Optional[Dict]:
        """获取详情"""
        raise NotImplementedError
    
    def download_comic(self, album_id: str, download_dir: str) -> Tuple[Dict, bool]:
        """下载内容"""
        raise NotImplementedError
    
    def get_image_url(self, album_id: str, page_num: int) -> str:
        """获取图片在线URL"""
        raise NotImplementedError
    
    def convert_to_standard_format(self, albums: List[Dict]) -> Dict[str, List]:
        """转换为系统标准格式"""
        raise NotImplementedError
```

### 6.4 新增平台接入步骤

#### 步骤 1：添加平台枚举

```python
# core/platform.py
class Platform(Enum):
    JM = "JM"
    PK = "PK"
    JAVDB = "JAVDB"
    YOUR_PLATFORM = "YP"  # 新增平台
```

#### 步骤 2：添加平台目录常量

```python
# core/constants.py
YP_PICTURES_DIR = "data/pictures/YP"
YP_COVER_DIR = "static/cover/YP"
YP_RECOMMENDATION_CACHE_DIR = "data/recommendation_cache/YP"
```

#### 步骤 3：创建适配器

```python
# third_party/your_platform_adapter.py
from third_party.base_adapter import BaseAdapter

class YourPlatformAdapter(BaseAdapter):
    def get_platform(self) -> Platform:
        return Platform.YOUR_PLATFORM
    
    # 实现其他必需方法...
```

#### 步骤 4：注册到工厂

```python
# third_party/adapter_factory.py
def get_adapter(platform: Platform, existing_tags: List = None):
    if platform == Platform.YOUR_PLATFORM:
        return YourPlatformAdapter(existing_tags)
    # ...
```

#### 步骤 5：添加子模块（可选）

```bash
git submodule add https://github.com/xxx/your-platform-lib.git comic_backend/third_party/your-platform-lib
```

## 8. 开发规范

### 8.1 代码风格

- **Python**：遵循 PEP 8 规范
- **JavaScript/Vue**：使用 ESLint + Prettier
- **注释**：使用中文注释，函数需有文档字符串

### 8.2 分支管理

```
main           # 主分支，稳定版本
  ├── develop  # 开发分支
  │     ├── feat/xxx   # 功能分支
  │     ├── fix/xxx    # 修复分支
  │     └── refactor/xxx # 重构分支
```

### 8.3 提交规范

```
feat: 新增功能
fix: 修复 Bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具链更新
```

### 8.4 新功能开发流程

```
1. 从 develop 创建功能分支
   git checkout -b feat/new-feature

2. 开发并编写测试
   - 遵循现有代码结构
   - 添加 E2E 或 Integration 测试

3. 运行测试门禁
   ./tests/run_test_gate.ps1

4. 提交代码
   git add .
   git commit -m "feat: 新增 XXX 功能"

5. 推送并创建 PR
   git push origin feat/new-feature

6. 合并到 develop
```

### 8.5 关键文件清单

| 文件 | 用途 |
|------|------|
| `comic_backend/app.py` | 后端入口，初始化配置、备份、清理 |
| `comic_backend/api/__init__.py` | 蓝图注册 |
| `comic_frontend/src/main.js` | 前端入口 |
| `comic_frontend/src/router/index.js` | 路由配置 |
| `comic_frontend/src/stores/index.js` | Store 注册 |
| `tests/conftest.py` | Pytest 配置和 fixtures |
| `tests/playwright.config.js` | Playwright 配置 |
