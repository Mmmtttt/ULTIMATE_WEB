# 数据库整理与视频本地导入全量设计书（漫画/视频双模式）

## 1. 目标与范围

本设计覆盖以下能力：

1. 统一“数据库整理”入口（已在 Mine 页使用 mode+action）。
2. 漫画模式整理能力（已实现）继续保留：
   - 修复封面
   - 按清洗标题去重（保留章节差异）
   - LOCAL 补全信息（JM 优先、失败回退 PK）
3. 新增视频模式本地导入能力：
   - 与漫画本地导入入口位置一致（Mine 页入口）
   - 仅服务端路径导入，且仅支持“文件夹”
   - 递归扫描视频文件，忽略压缩包
4. 新增视频模式数据库整理能力：
   - 按 code 去重（本地库 + 预览库）
   - LOCAL 补全信息（按 code 查 JAVDB/JAVBUS 并回写）
5. 视频详情页新增本地文件播放器，支持播放本地导入源视频。

---

## 2. 设计原则

1. 统一编排：所有整理动作继续通过 `/api/v1/organize/options` 与 `/api/v1/organize/run` 调度。
2. 模式隔离：漫画与视频动作清单可不同，互不影响。
3. 可扩展：视频动作以 action 注册方式接入，后续可继续新增。
4. 幂等安全：
   - 去重仅移入回收站，不做永久删除
   - LOCAL 补全成功后打标，下次自动跳过
5. 复用优先：复用现有 `video_app_service` 中封面/预览图/预览视频本地缓存与下载链路。

## 3. 前端设计

## 3.1 Mine 入口

- 漫画模式：保持 `本地漫画导入`。
- 视频模式：新增 `本地视频导入`，入口位置与漫画一致（同分组 cell）。

## 3.2 整理数据库弹窗

视频模式 options 改为可执行动作：

1. `deduplicate_by_code`
   - 名称：查重并移入回收站
2. `enrich_local_metadata`
   - 名称：LOCAL 补全信息

漫画模式保持现有 3 个动作不变。

## 3.3 视频本地导入页（新页面）

页面职责：

1. 输入“服务端文件夹绝对路径”。
2. 调用后端导入接口并显示结果：
   - 扫描文件数
   - 成功导入数
   - 跳过数（code 冲突/非法文件）
   - 失败数
3. 导入完成后刷新视频列表（本地库）。

交互限制说明：

1. 仅支持目录路径，不支持压缩包路径。
2. 压缩包文件即使在目录中被扫描到也会忽略。

## 3.4 视频详情页本地播放器

在现有播放区基础上新增“本地文件播放区”：

1. 若 `local_video_path` 存在，则显示 `<video controls>`。
2. 本地播放区优先用于本地导入视频回放。
3. 与“预览视频”（抓取到的短预览）并存，不互斥。

## 4. 后端架构设计

## 4.1 DatabaseOrganizeService 扩展

`mode=video` 动作从占位改为真实动作：

1. `deduplicate_by_code` -> `VideoAppService.organize_deduplicate_by_code()`
2. `enrich_local_metadata` -> `VideoAppService.organize_enrich_local_metadata()`

## 4.2 Video 数据模型扩展

`domain/video/entity.py` 增加字段：

1. `local_video_path: str = ""`
   - 指向本地源视频静态路径（`/media/video/LOCAL/<video_id>/source.ext`）
2. `local_source_path: str = ""`
   - 导入时的原始文件绝对路径（用于审计/排障）
3. `local_metadata_enriched: bool = False`
   - LOCAL 补全信息幂等标志

`from_dict/to_dict` 同步该字段，避免持久化丢失。

## 4.3 存储目录规范

视频 LOCAL 统一目录：

1. 根目录：`data/video/LOCAL`
2. 每条视频独立目录：`data/video/LOCAL/<video_id>/`
3. 本地源视频文件：`source.<原后缀>`（如 `source.mp4`）
4. 补全下载资源沿用现有缓存方法写入同目录：
   - `cover.jpg`
   - `thumbs/thumb-xxxx.jpg`
   - `preview.*` 或 `hls/index.m3u8`

## 5. 视频本地导入详细设计

## 5.1 接口

新增：

- `POST /api/v1/video/local-import/from-path`

请求：

```json
{
  "source_path": "D:\\Videos\\Local"
}
```

响应（示例）：

```json
{
  "scanned_files": 28,
  "imported_count": 12,
  "skipped_count": 14,
  "failed_count": 2,
  "imported_ids": ["LOCALV_xxx", "..."],
  "summary": "导入完成：成功 12，跳过 14，失败 2"
}
```

## 5.2 扫描规则

1. `source_path` 必须存在且是目录。
2. 递归扫描所有文件。
3. 只处理常见视频后缀（不区分大小写）：
   - `.mp4 .mkv .avi .mov .wmv .flv .webm .m4v .ts .m2ts .rmvb .mpg .mpeg`
4. 压缩包后缀一律忽略：
   - `.zip .rar .7z .tar .gz .bz2 .xz`

## 5.3 code 提取规则

从文件名（不含扩展名）提取，优先级：FC2 > 普通番号 > 异常 code。

### A. 普通番号

匹配：

1. 2~7 位字母 + 2~4 位数字
2. 中间允许 `_` / `-` / 空格 / 无分隔
3. 大小写不敏感

标准化输出：

- 字母转大写
- 统一为 `AAA-123` 形式

### B. FC2 番号

匹配（大小写不敏感）：

1. 前缀 `fc2`
2. 可包含 `ppv` 字样
3. 后接 4~8 位数字（允许中间 `-` / `_` / 空格）

标准化输出：

- 统一为 `FC2-PPV-<数字>`

### C. 异常 code（兜底唯一标识）

当文件名中无法提取 A/B 规则 code 时：

1. 生成不符合 A/B 的 code（示例：`LOCALERR_<uuid8>`）
2. 保证全库唯一，冲突则重试

## 5.4 导入入库规则

每个视频文件：

1. 生成 `video_id`（前缀 `LOCALV_`）。
2. 复制到 `video/LOCAL/<video_id>/source.<ext>`。
3. 写入视频库记录：
   - `id/title/code/local_video_path/local_source_path/...`
   - `title` 默认文件名
   - `creator/actors/tag_ids` 初始可为空
4. 以规范化 code 做冲突检查：
   - 若本地库已有同 code（规范化后一致）则跳过

## 6. 视频整理动作设计

## 6.1 动作一：按 code 去重（`deduplicate_by_code`）

作用范围：

1. 本地库 `videos_database.json/videos`
2. 预览库 `video_recommendations_database.json/video_recommendations`
3. 仅 `is_deleted != true` 记录

判重规则：

1. 使用 code 规范化（去分隔符+大写）
2. 规范化后相同视为重复

处理策略：

1. 每组保留首条
2. 其余置 `is_deleted=true`（移入回收站）

返回：

1. `home.moved_to_trash`
2. `recommendation.moved_to_trash`
3. `summary`

## 6.2 动作二：LOCAL 补全信息（`enrich_local_metadata`）

候选范围：

1. 本地库中 `id` 以 `LOCAL` / `LOCALV` 开头
2. `is_deleted != true`
3. `local_metadata_enriched != true`
4. `code` 非空

匹配流程（按用户要求）：

1. 按 code 先查 JAVDB
2. JAVDB 无结果再查 JAVBUS
3. 每个平台仅取**第一条搜索结果**直接视为匹配，不再二次比对
4. 两端都无结果则跳过

回写字段：

1. `title`
2. `creator`（默认取演员首位）
3. `actors`
4. `cover_path`（远程）
5. `thumbnail_images`（远程）
6. `preview_video`（远程）
7. `tag_ids`（标签映射/新建）
8. `local_metadata_enriched=true`

资源落盘：

1. 封面 -> 复用 `cache_cover_to_static_async`（并保持 local 字段）
2. 预览图 -> 复用 `cache_thumbnail_images_async(..., source=local, force=True)`
3. 预览视频 -> 复用 `cache_preview_video_async(..., source=local, force=True)`
4. 资产统一落入 `video/LOCAL/<video_id>/...`

标签处理：

1. 读取视频标签库（`content_type=video`）
2. 远端 tag 不存在则创建
3. 追加绑定到 `video.tag_ids`（去重）

返回统计：

1. `processed_candidates`
2. `matched_on_javdb`
3. `matched_on_javbus`
4. `updated_records`
5. `updated_tag_bindings`
6. `created_tags`
7. `skipped_no_match`
8. `skipped_already_enriched`
9. `summary`

## 8. 测试设计（集成）

## 8.1 视频整理 options

断言 `mode=video` 返回：

1. `deduplicate_by_code`（implemented=true）
2. `enrich_local_metadata`（implemented=true）

## 8.2 视频去重

构造本地库/预览库重复 code（含大小写与分隔差异），运行动作后断言：

1. 重复项仅 `is_deleted=true`
2. 保留项仍可用
3. 统计值正确

## 8.3 视频 LOCAL 补全

mock `video_api.get_video_adapter`：

1. JAVDB 空结果 -> 回退 JAVBUS 命中
2. 仅使用第一条结果
3. 成功回写标题/作者/标签/幂等标志

## 8.4 视频本地导入

构造临时目录树（含视频文件、压缩包、非视频文件）：

1. 仅视频被导入
2. 压缩包被忽略
3. 能提取常规 code 与 FC2
4. 无 code 文件生成 `LOCALERR_*`
5. `local_video_path` 指向 `/media/video/LOCAL/<id>/source.ext`
