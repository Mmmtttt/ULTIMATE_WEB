# 随机流推荐算法扩展说明

## 1. 功能入口
- 后端服务：`comic_backend/application/random_feed_service.py`
- 接口：
  - `POST /api/v1/feed/session` 创建会话（随机序列）
  - `GET /api/v1/feed/items` 拉取下一批推荐项
  - `POST /api/v1/feed/session/refresh` 刷新序列
  - `GET /api/v1/feed/strategies` 查看可用策略

## 2. 当前默认策略
- 默认策略名：`random_priority_v1`
- 目标：`随机优先`，轻度引入业务权重。
- 当前内置信号：
  - 评分（`score`）轻度加权
  - 阅读进度（未读略高）
  - 来源（`preview` 轻度降权）

## 3. 策略扩展点
`random_feed_service.py` 中提供了统一策略接口：

```python
class FeedWeightStrategy:
    name = "base"
    def weight(self, candidate: FeedWorkCandidate) -> float:
        return 1.0
```

候选字段 `FeedWorkCandidate` 已包含：
- `mode`：`comic`/`video`
- `source`：`local`/`preview`
- `content_id`
- `title`、`author`
- `score`
- `tag_ids`
- `total_units`、`current_unit`
- `page_numbers`（漫画）
- `image_urls`（视频）

## 4. 如何新增策略
1. 在 `random_feed_service.py` 新增策略类（继承 `FeedWeightStrategy`）。
2. 实现 `weight(candidate)`，返回 `>0` 的权重值。
3. 在 `RandomFeedService.__init__` 中注册：
   - `self.register_strategy("your_strategy_name", YourStrategyClass)`
4. 前端创建会话时传 `strategy_name` 即可启用。

示例：

```python
class ScoreTagBoostStrategy(FeedWeightStrategy):
    name = "score_tag_boost_v1"

    def weight(self, candidate: FeedWorkCandidate) -> float:
        w = 1.0
        w += max(0.0, min(12.0, float(candidate.score or 0.0))) / 12.0 * 0.4
        if "tag_action" in (candidate.tag_ids or []):
            w += 0.2
        return max(0.05, w)
```

## 5. 设计约束
- 采样方式为**有放回随机抽样**，保证“无限下滑、允许重复”。
- 权重仅影响抽样概率，不影响会话结构。
- 刷新序列会重置随机种子与游标，重新生成序列。

## 6. 推荐新增信号
- 用户侧：
  - 已读/未读
  - 最近阅读时间衰减
  - 收藏作者
  - 标签偏好（包含/排除）
- 内容侧：
  - 评分区间
  - 标签覆盖度
  - 来源（本地/预览）
