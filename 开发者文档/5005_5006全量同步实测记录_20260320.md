- 两端实际运行的代码版本不一致（接口行为不一致，导致“只同步元数据，不同步媒体”）。

## 3. 处理步骤
1. 修复本机 `conda` 激活脚本错误（避免每次命令被环境脚本报错污染）。
2. 确认 `5005/5006` 健康检查均可达：`GET /health` 返回 200。
3. 停止旧进程并重新启动两端后端，确保都从各自目录的最新源码启动。
4. 先在空目标目录做一次完整传输验证（`data_sync_target_empty`），确认资产链路可用。
5. 再恢复目标端默认数据目录 `./comic_backend/data`，再次执行完整传输验证。

## 4. 实际执行命令（核心）

```powershell
# 在源端目录执行
python scripts/e2e_sync_transfer.py `
  --local-base http://127.0.0.1:5005 `
  --remote-base http://127.0.0.1:5006 `
  --source-data-root D:\code\临时并行开发目录\ULTIMATE_WEB\comic_backend\data `
  --target-data-root D:\code\临时并行开发目录\第二\ULTIMATE_WEB\comic_backend\data `
  --wait-seconds 10
```

该脚本会自动执行：
- 5006 创建邀请 `pairing/invite`
- 5005 连接配对 `pairing/connect`
- 5005 执行 `directional/preview`（检查待传输的数据与资产）
- 5005 执行 `directional/push`
- 对比传输前后目录文件数与大小
- 目标端媒体 URL 抽样探测（图片、m3u8、缩略图等）
