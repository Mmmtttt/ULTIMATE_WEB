# 三平台可执行文件构建手册

更新时间：2026-03-19

本文面向新开发者，目标是：
- 克隆仓库后能快速完成环境准备
- 修改代码后可按统一命令构建
- 最终产出三平台包：Windows `.exe`、Linux 可执行文件、Android `.apk`

## 1. 先说结论

1. Windows 本机可以直接构建：
- Windows 可执行文件
- Android APK

2. Linux 可执行文件必须在 Linux 环境构建：
- 本地 Linux 主机
- 或 CI 的 Ubuntu Runner（推荐）

3. “一次触发产出三平台”推荐用 CI：
- 工作流文件：`.github/workflows/release-three-platforms.yml`

## 2. 从零开始：克隆仓库

```powershell
git clone <你的仓库地址>
cd ULTIMATE_WEB
git submodule update --init --recursive
```

## 3. Windows 环境准备（Conda base）

在 `base` 环境执行一键安装脚本：

```powershell
conda activate base
powershell -ExecutionPolicy Bypass -File .\scripts\install_build_env_windows_base.ps1
```

说明：
- 脚本默认会复用本机已安装的 Android cmdline-tools，不会每次重复下载
- 若你确实要强制重下，使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\install_build_env_windows_base.ps1 -RefreshCmdlineTools
```

安装完成后新开终端，验证：

```powershell
conda activate base
python --version
node --version
npm --version
npx --version
java --version
sdkmanager.bat --version
adb --version
python -c "import importlib.util; print('PyInstaller=', bool(importlib.util.find_spec('PyInstaller')), 'flask_cors=', bool(importlib.util.find_spec('flask_cors')))"
```

## 4. 日常开发流程（修改代码）

```powershell
git checkout -b feat/<你的分支名>
# 修改前后端代码
git add .
git commit -m "feat: <你的变更说明>"
```

## 5. 本地构建（Windows 主机）

### 5.1 一条命令构建 Windows + Android

```powershell
conda activate base
python .\scripts\release_unified.py --targets "windows,android" --execute --build-output output\multi_target --package-output output\packages
```

### 5.2 本地产物位置

- Windows EXE：
`output/packages/windows/dist/ultimate_backend_windows.exe`

- Windows 发布包内副本：
`output/packages/windows/release_bundle/bin/ultimate_backend_windows.exe`

- Android APK：
`output/packages/android/apk/app-debug.apk`

## 6. Linux 可执行文件构建（Linux 主机）

在 Linux 环境执行：

```bash
git clone <你的仓库地址>
cd ULTIMATE_WEB
git submodule update --init --recursive

python3 -m pip install --upgrade pip
python3 -m pip install -r comic_backend/requirements.txt pyinstaller
npm ci --prefix comic_frontend

python3 scripts/release_unified.py --targets linux --execute --build-output output/multi_target --package-output output/packages
```

产物位置：
- `output/packages/linux/dist/ultimate_backend_linux`
- `output/packages/linux/release_bundle/bin/ultimate_backend_linux`

## 7. 一次产出三平台（推荐：CI）

### 7.1 触发构建

```bash
git checkout -b feat/<你的分支名>
# 修改代码后
git add .
git commit -m "feat: <你的变更说明>"
git push origin feat/<你的分支名>

gh auth login
gh workflow run release-three-platforms.yml --ref feat/<你的分支名>
RUN_ID=$(gh run list --workflow release-three-platforms.yml --branch feat/<你的分支名> --limit 1 --json databaseId --jq '.[0].databaseId')
gh run watch "$RUN_ID"
```

### 7.2 下载三平台产物

```bash
mkdir -p output/from_ci
gh run download "$RUN_ID" -n ultimate-release-bundle -D output/from_ci
```

下载后产物位置：
- Windows EXE：`output/from_ci/ultimate-windows/dist/ultimate_backend_windows.exe`
- Linux 可执行文件：`output/from_ci/ultimate-linux/dist/ultimate_backend_linux`
- Android APK：`output/from_ci/ultimate-android/apk/app-debug.apk`
- 哈希清单：`output/from_ci/RELEASE_MANIFEST_SHA256.txt`

## 8. 常见问题

1. Android 在 Windows 下报“路径包含非 ASCII 字符”：
- 当前打包脚本已自动将 Android 构建工作目录迁移到英文路径（`%LOCALAPPDATA%\UltimateWebBuild\android_workspace\...`），无需手动改仓库路径。

2. 构建 Android 时 Java 版本报错：
- 需使用 JDK 21。
- 脚本已按 JDK 21 设计；若你本机 PATH 仍指向旧版本，重跑安装脚本并重新打开终端。

3. `sdkmanager` 命令找不到：
- Windows 下优先使用 `sdkmanager.bat --version` 验证。
