# 三平台可执行文件构建手册

更新时间：2026-03-19

本文档面向新开发者，目标是：
- 从零克隆仓库后，快速完成环境准备
- 在统一代码库中开发新功能（默认对平台无感知）
- 通过统一流水线产出三类发布包：Windows、Linux、Android

## 1. 当前架构结论

1. Windows 主机可直接构建：Windows EXE + Android APK。
2. Linux 可执行文件需要在 Linux 主机或 Linux CI Runner 构建。
3. 若要“一次触发得到三平台产物”，推荐用 CI 工作流统一编排。
4. Android 现在已支持：APK 启动时拉起本地 Python 后端，前端默认连 `127.0.0.1:5000`，数据目录写入应用私有可写路径。

## 2. 从零开始：克隆仓库

```powershell
git clone <你的仓库地址>
cd ULTIMATE_WEB
git submodule update --init --recursive
```

## 3. Windows（Conda base）一键安装构建环境

在 Windows PowerShell 中执行：

```powershell
conda activate base
powershell -ExecutionPolicy Bypass -File .\scripts\install_build_env_windows_base.ps1
```

安装完成后，重新打开终端并验证：

```powershell
conda activate base
python --version
node --version
npm --version
npx --version
java --version
sdkmanager.bat --version
adb --version
python -c "import importlib.util; print('PyInstaller=', bool(importlib.util.find_spec('PyInstaller')), 'flask_cors=', bool(importlib.util.find_spec('flask_cors')))"
```

## 4. 日常开发（统一代码库）

```powershell
git checkout -b feat/<你的分支名>
# 修改前后端代码
git add .
git commit -m "feat: <变更说明>"
```

说明：
- 后续开发默认改业务代码，不需要为 Android/Windows/Linux 写三套逻辑。
- 平台差异由统一构建脚本在打包阶段注入（例如 Android 的嵌入式后端启动链路）。

## 5. Windows 主机本地出包（Windows + Android）

### 5.1 一条命令执行构建+打包

```powershell
conda activate base
python .\scripts\release_unified.py --targets "windows,android" --execute --build-output output\multi_target --package-output output\packages
```

### 5.2 产物位置

- Windows 主产物：`output/packages/windows/dist/ultimate_backend_windows.exe`
- Windows 发布包副本：`output/packages/windows/release_bundle/bin/ultimate_backend_windows.exe`
- Android 产物：`output/packages/android/apk/app-debug.apk`
- 打包汇总：`output/packages/packaging_summary.json`

## 6. Linux 可执行文件构建（Linux 主机或 Linux CI）

在 Linux 环境执行：

```bash
git clone <你的仓库地址>
cd ULTIMATE_WEB
git submodule update --init --recursive

python3 -m pip install --upgrade pip
python3 -m pip install -r comic_backend/requirements.txt pyinstaller
npm ci --prefix comic_frontend

python3 scripts/release_unified.py --targets linux --execute --build-output output/multi_target --package-output output/packages
```

产物位置：
- `output/packages/linux/dist/ultimate_backend_linux`
- `output/packages/linux/release_bundle/bin/ultimate_backend_linux`

## 7. 一次触发三平台（推荐 CI）

### 7.1 触发工作流

```bash
git checkout -b feat/<你的分支名>
# 修改代码
git add .
git commit -m "feat: <变更说明>"
git push origin feat/<你的分支名>

gh auth login
gh workflow run release-three-platforms.yml --ref feat/<你的分支名>
RUN_ID=$(gh run list --workflow release-three-platforms.yml --branch feat/<你的分支名> --limit 1 --json databaseId --jq '.[0].databaseId')
gh run watch "$RUN_ID"
```

### 7.2 下载三平台产物

```bash
mkdir -p output/from_ci
gh run download "$RUN_ID" -n ultimate-release-bundle -D output/from_ci
```

下载后常见路径：
- `output/from_ci/ultimate-windows/dist/ultimate_backend_windows.exe`
- `output/from_ci/ultimate-linux/dist/ultimate_backend_linux`
- `output/from_ci/ultimate-android/apk/app-debug.apk`
- `output/from_ci/RELEASE_MANIFEST_SHA256.txt`

## 8. Android 关键说明（当前已落地）

1. APK 内后端启动
- 打包脚本会自动向 Capacitor 生成的 Android 工程注入 Chaquopy。
- `MainActivity` 在 `onCreate` 启动 Python 线程，拉起后端服务。

2. 前端默认连本地后端
- 打包阶段会注入运行时 API 基地址：`http://127.0.0.1:5000/api`（可在 `build/packagers.json` 改端口）。

3. 数据目录可写
- Android 运行时通过 `BACKEND_RUNTIME_PROFILE=android` + `ANDROID_APP_FILES_DIR`，将数据落到应用私有可写目录。

4. 第三方能力开关
- 默认 `android_backend_enable_third_party=false`，即 APK 内后端默认关闭第三方平台能力。
- 如需开启，修改 `build/packagers.json` 后重打包。

## 9. 常见问题

1. `SDK location not found`
- 统一打包脚本会在 Gradle 前自动写入 `android/local.properties`。
- 若仍失败，先确认：`sdkmanager.bat --version`。

2. Android 构建工作目录锁文件（`.gradle/.../checksums.lock`）
- 脚本已改为每次输出目录对应独立 workspace，避免旧锁复用。
- 如极端情况下仍遇到锁占用，可先结束 `java.exe` 后重试。

3. PowerShell 每次出现 conda 激活脚本解析报错
- 错误来源通常是 `D:\environment\conda\etc\conda\activate.d\android_sdk_env.ps1` 内容被写成单行。
- 请将该文件修正为多行合法 PowerShell 脚本（你当前环境已修复过一次）。
