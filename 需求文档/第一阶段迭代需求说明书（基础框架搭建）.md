## 1. 迭代核心目标
1. 完成前后端项目初始化，搭建本地JSON文件存储结构，实现JSON文件读写工具；
2. 实现「漫画图片文件夹准备→元数据写入JSON→主页列表展示→详情页跳转→阅读页基础浏览」的完整核心链路；

4. 实现基础的阅读进度记忆、图片懒加载功能，保证核心浏览体验可用；
5. 完成基础日志体系搭建，保证开发阶段问题可定位、可追溯。

## 2. 功能需求详情
### 2.1 后端功能需求
#### 2.1.1 本地存储结构初始化（必须实现）
1. 创建 `static/cover/` 目录，用于存放漫画封面缩略图。缩略图就是漫画第一章的图片。

#### 2.1.2 核心工具类开发（必须实现）
1. **JSON文件读写工具**（`utils/json_handler.py`）：
   - 实现 `read_json()` 方法：读取 `comics_database.json` 文件，返回Python字典；
   - 实现 `write_json(data)` 方法：将Python字典写入 `comics_database.json` 文件，覆盖原有内容，保证原子性写入（避免文件损坏）；
   - 实现 `generate_id(prefix)` 方法：生成唯一ID，格式为 `prefix_时间戳`，如 `comic_1709000000`。
2. **文件解析工具**（`utils/file_parser.py`）：
   - 实现 `parse_comic_images(comic_id)` 方法：读取 `data/pictures/{comic_id}/` 目录下的图片文件，按文件名自然排序，返回图片路径数组；
   - 实现 `validate_image_format(file_path)` 方法：校验图片格式是否为jpg/png/webp，过滤无效文件；
   - 实现 `natural_sort(file_list)` 方法：对文件名进行自然排序，保证 `1.jpg、2.jpg、10.jpg` 按数字顺序排列。
3. **图片处理工具**（`utils/image_handler.py`）：
   - 实现 `generate_cover(comic_id, first_image_path)` 方法：读取漫画第一张图片，压缩生成封面缩略图，保存到 `static/cover/{comic_id}.jpg`，返回封面路径；
   - 实现 `get_image_stream(comic_id, page_num)` 方法：流式返回指定页码的图片资源，避免内存溢出。

#### 2.1.3 核心接口开发（必须实现）
| 接口名称 | 请求方式 | 接口地址 | 核心参数 | 核心功能 |
|----------|----------|----------|----------|----------|
| 漫画初始化接口 | POST | /api/v1/comic/init | comic_id（必填）、title（必填） | 1. 校验 `data/pictures/{comic_id}/` 目录是否存在；<br>2. 解析目录下的图片，计算总页数；<br>3. 生成封面缩略图；<br>4. 生成漫画元数据，写入 `comics_database.json` 的 `comics` 数组；<br>5. 自动初始化阅读进度（current_page=1） |
| 漫画列表查询接口 | GET | /api/v1/comic/list | 无 | 读取 `comics_database.json`，返回所有漫画的id、title、cover_path、total_page、current_page、last_read_time |
| 漫画详情基础接口 | GET | /api/v1/comic/detail | comic_id（必填） | 读取JSON文件，返回指定漫画的id、title、cover_path、total_page、current_page、last_read_time |
| 漫画图片列表接口 | GET | /api/v1/comic/images | comic_id（必填） | 调用文件解析工具，返回指定漫画的所有图片路径数组，按阅读顺序排序 |
| 单张图片读取接口 | GET | /api/v1/comic/image | comic_id（必填）、page_num（必填） | 调用图片处理工具，流式返回指定页码的图片资源 |
| 阅读进度保存接口 | PUT | /api/v1/comic/progress | comic_id（必填）、current_page（必填） | 读取JSON文件，更新指定漫画的current_page和last_read_time，写入JSON文件 |

### 2.2 前端功能需求
#### 2.2.1 基础架构与路由（必须实现）
1. 完成Vue3项目初始化，配置Vue Router、Axios、pinia状态管理；
2. 完成底部双页签导航栏开发：「主页」、「我的页面」，我的页面本期仅做占位，显示「功能开发中」；
3. 完成路由配置，定义4个核心页面路由：
   | 页面名称 | 路由地址 | 说明 |
   |----------|----------|------|
   | 主页 | /home | 默认首页 |
   | 我的页面 | /mine | 占位页 |
   | 漫画详情页 | /detail/:comicId | 动态路由，携带漫画ID |
   | 漫画阅读页 | /reader/:comicId | 动态路由，携带漫画ID |
4. 完成全局响应式样式配置，定义PC端/手机端断点，保证页面元素自动适配。

#### 2.2.2 主页开发（必须实现）
1. 页面结构：顶部搜索框（本期仅做UI占位，不实现搜索功能）、中间漫画列表区域、底部页签导航；
2. 核心功能：
   - 调用漫画列表接口，渲染漫画缩略卡片，PC端4列布局，手机端单列布局；
   - 漫画卡片展示内容：封面图、漫画标题、总页数、「已读X/总页数」进度；
   - 支持上下滑动浏览漫画列表，滚动加载流畅，无卡顿；
   - 点击漫画卡片，跳转至对应漫画的详情页。

#### 2.2.3 漫画详情页（简化版，必须实现）
1. 页面结构：顶部返回按钮、中间漫画封面区域、底部基础信息区域、「进入阅读」固定按钮；
2. 核心功能：
   - 根据路由中的漫画ID，调用详情接口，渲染漫画信息；
   - 展示内容：漫画封面、标题、总页数、当前阅读进度、最后阅读时间；
   - 点击封面图/「进入阅读」按钮，跳转至漫画阅读页；
   - 点击返回按钮，返回主页。

#### 2.2.4 漫画阅读页（基础版，必须实现）
1. 页面结构：全屏漫画展示区域、顶部信息栏、底部操作栏；
2. 核心功能：
   - 根据漫画ID，调用图片列表接口，获取图片路径数组；
   - 基础翻页功能：
     - PC端：点击页面左半区域/按左方向键→上一页，点击页面右半区域/按右方向键/空格→下一页，按ESC键返回详情页；
     - 手机端：左右触屏滑动翻页，左滑→下一页，右滑→上一页；
   - 进度展示：顶部栏实时显示「当前页/总页数」，页码与翻页同步更新；
   - 阅读进度记忆：翻页时自动调用进度保存接口，退出页面时保存最终进度，下次进入自动定位到上次阅读的页码；
   - 图片懒加载：默认预加载当前页+前后3张图片，仅渲染当前可视区域图片，减少内存占用；
   - 基础适配：图片默认适配屏幕宽度，高度自适应，超出屏幕可上下滑动查看。

#### 2.2.5 响应式适配（必须实现）
1. 所有页面适配PC端主流分辨率（1920*1080、2560*1440）；
2. 所有页面适配手机端主流机型（375px、414px宽度），横竖屏切换无样式错乱；
3. 操作按钮适配触屏操作，手机端按钮尺寸不小于44px*44px，保证点击精准度。

## 3. 非功能需求
1. 性能要求：漫画列表首屏加载时间≤500ms，单张图片加载时间≤200ms，翻页操作响应时间≤200ms；
2. 稳定性要求：连续翻页100次无崩溃、无内存溢出，JSON文件读写失败不影响服务运行，文件损坏时自动备份上一版本；
3. 兼容性要求：PC端支持Chrome、Edge、Firefox浏览器，手机端支持Chrome移动端、微信内置浏览器。

## 4. 日志需求
### 4.1 后端日志要求
| 日志级别 | 记录场景 | 必须记录的内容 |
|----------|----------|----------------|
| INFO | 正常操作 | 漫画初始化成功（漫画ID、标题、总页数）、阅读进度保存成功（漫画ID、页码）、JSON文件读写成功、接口正常请求完成 |
| ERROR | 异常操作 | 漫画初始化失败（失败原因、漫画ID、图片目录路径）、图片解析失败（漫画ID、页码、错误原因）、JSON文件读写失败（错误堆栈）、接口500异常（请求参数、错误堆栈） |
| WARN | 警告场景 | 图片格式不支持、图片目录内无有效图片、请求参数非必填项缺失 |

### 4.2 前端日志要求
1. 控制台输出所有接口请求的参数、返回值、错误信息；
2. 图片加载失败时，输出图片路径、错误原因；
3. 路由跳转异常、翻页操作异常时，输出错误堆栈信息；
4. 所有日志按「时间-模块-内容」格式输出，便于排查问题。
