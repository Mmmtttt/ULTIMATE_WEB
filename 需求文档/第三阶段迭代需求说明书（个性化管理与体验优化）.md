## 2. 功能需求详情
### 2.1 后端功能需求
#### 2.1.1 JSON文件结构最终扩展（必须实现）
在 `comics_database.json` 中完善 `lists`、`comics[].list_ids`、`user_config` 字段，结构遵循项目介绍文档中的约定。
#### 2.1.2 新增核心接口开发（必须实现）
| 接口名称 | 请求方式 | 接口地址 | 核心参数 | 核心功能 |
|----------|----------|----------|----------|----------|
| 清单管理接口-新增 | POST | /api/v1/list/add | list_name（必填）、desc（可选） | 1. 生成唯一清单ID；<br>2. 新增清单到JSON的 `lists` 数组 |
| 清单管理接口-列表 | GET | /api/v1/list/list | 无 | 读取JSON，返回所有清单的id、list_name、desc、漫画数量（统计 `comics` 中包含该list_id的数量）、create_time |
| 清单管理接口-详情 | GET | /api/v1/list/detail | list_id（必填） | 读取JSON，返回清单基础信息+清单内的漫画列表（筛选 `comics` 中包含该list_id的漫画） |
| 清单管理接口-编辑 | PUT | /api/v1/list/edit | list_id（必填）、list_name（可选）、desc（可选） | 更新JSON中对应清单的对应字段 |
| 清单管理接口-删除 | DELETE | /api/v1/list/delete | list_id（必填） | 1. 从JSON的 `lists` 数组中删除清单；<br>2. 从所有漫画的 `list_ids` 数组中移除该list_id |
| 漫画清单绑定接口 | PUT | /api/v1/list/comic/bind | list_id（必填）、comic_id_list（漫画ID数组，必填） | 更新JSON中对应漫画的 `list_ids` 数组，加入该list_id |
| 漫画清单移出接口 | DELETE | /api/v1/list/comic/remove | list_id（必填）、comic_id_list（漫画ID数组，必填） | 更新JSON中对应漫画的 `list_ids` 数组，移除该list_id |
| 配置查询接口 | GET | /api/v1/config | 无 | 读取JSON的 `user_config` 字段，无配置则返回默认值（m=3、左右翻页、白色背景） |
| 配置保存接口 | PUT | /api/v1/config | preload_num（可选）、default_page_mode（可选）、default_background（可选） | 更新JSON的 `user_config` 字段，支持单字段/全字段更新 |
| 漫画列表接口优化 | GET | /api/v1/comic/list | sort_type（可选：create_time/score/read_time） | 读取JSON，按指定字段排序后返回漫画列表 |

#### 2.1.3 工具类最终优化（必须实现）
1. 优化图片预加载逻辑，支持根据自定义m值动态调整预加载范围；
2. 优化上下翻页模式的图片拼接处理，保证长图无缝衔接；
3. 新增批量操作工具，支持漫画批量绑定标签、批量加入清单。

### 2.2 前端功能需求
#### 2.2.1 我的页面（完整版，必须实现）
1. 页面结构：
   - 顶部：页面标题「我的」；
   - 功能入口区：4个核心功能卡片，分别为「我的清单」、「标签管理」、「我的收藏」、「系统配置」；
   - 数据概览区：显示漫画总数、已读漫画数、标签总数、清单总数；
   - 底部：页签导航栏。
2. 核心功能模块：
   - 我的清单：
     1. 展示所有自定义清单列表，显示清单名称、简介、漫画数量；
     2. 支持新增清单、编辑清单、删除清单（二次确认）；
     3. 点击清单，进入清单详情页，展示清单内的所有漫画，支持移出漫画、批量加入漫画；
     4. 漫画卡片/详情页新增「加入清单」按钮，支持选择多个清单，一键加入。
   - 标签管理：
     1. 复用第二阶段的标签管理功能，优化UI布局，适配PC/手机端；
     2. 新增标签批量绑定功能，支持选择多个漫画，批量绑定同一标签。
   - 我的收藏：
     1. 内置「我的收藏」默认清单，不可删除；
     2. 漫画卡片/详情页新增「收藏/取消收藏」按钮，一键加入/移出收藏清单；
     3. 我的页面点击「我的收藏」，直接进入收藏清单详情页。
   - 系统配置：
     1. 预加载数量配置：数字输入框，范围1-10，默认值3，支持用户自定义预加载图片数量m，修改后实时保存；
     2. 默认翻页模式配置：单选框，选项为「左右翻页」、「上下翻页」，修改后下次进入阅读页自动生效；
     3. 默认阅读背景配置：单选框，选项为「白色背景」、「深色背景」、「护眼色背景」，修改后实时生效；
     4. 配置重置按钮：一键恢复所有配置为默认值，二次确认后生效。

#### 2.2.2 主页最终优化（必须实现）
1. 新增排序功能：顶部搜索栏旁新增排序按钮，支持按「添加时间」、「评分从高到低」、「最后阅读时间」排序；
2. 优化筛选功能：支持「评分区间筛选」，可选择最低分、最高分，筛选对应评分范围的漫画；
3. 优化漫画卡片：新增「收藏」标识，已收藏的漫画显示收藏图标；
4. 优化批量操作：支持长按进入批量选择模式，批量选择漫画，进行「加入清单」、「绑定标签」、「删除」操作。

#### 2.2.3 漫画阅读页（最终版，必须实现）
1. 翻页模式终极优化：
   - 新增上下翻页模式，支持与左右翻页模式一键切换；
   - 上下翻页模式实现图片无缝垂直拼接，视觉上为一张完整长图，滑动流畅无断层、无迟滞感；
   - 上下翻页模式支持滚轮滑动、触屏上下滑动，自动记录阅读位置；
   - 翻页模式切换后，自动保存到用户配置，下次进入默认使用该模式。
2. 自定义懒加载功能：
   - 读取用户配置的预加载数量m，浏览第n张图片时，自动预加载n+m张图片；
   - 预加载图片仅加载资源，不渲染，不阻塞当前页面操作；
   - 支持阅读页内临时修改预加载数量，实时生效。
3. 阅读体验全量优化：
   - 背景色切换：支持在阅读页内一键切换白色/深色/护眼色背景，修改后同步保存到用户配置；
   - 操作栏智能隐藏：点击屏幕非图片区域，显示/隐藏顶部、底部操作栏，最大化阅读区域；
   - 页码快速跳转：支持输入页码，一键跳转到指定页，超出总页数时给出提示；
   - 快捷键全适配：
     | 快捷键 | 功能 |
     |--------|------|
     | ← / A | 上一页 |
     | → / D / 空格 | 下一页 |
     | ↑ / W | 上滑（上下模式） |
     | ↓ / S | 下滑（上下模式） |
     | + / = | 放大图片 |
     | - / _ | 缩小图片 |
     | 0 | 重置缩放 |
     | Esc | 退出阅读页 |
     | B | 切换背景色 |
     | M | 切换翻页模式 |
   - 缩放优化：缩放后自动调整图片位置，保证图片不超出屏幕边界，拖动流畅无卡顿。
4. 进度记忆终极优化：
   - 自动记忆当前漫画的翻页模式、背景色、缩放比例、阅读页码；
   - 退出后重新进入，100%还原上次的阅读状态，无需用户手动调整。

#### 2.2.4 移动端全量适配优化（必须实现）
1. 全页面触屏操作优化：所有按钮、输入框适配触屏操作，点击无延迟、无误触；
2. 手势操作优化：
   - 所有页面支持侧滑返回上一页，符合移动端操作习惯；
   - 阅读页滑动灵敏度优化，滑动翻页无误触、无卡顿；
   - 捏合缩放优化，缩放精度提升，无缩放跳变；
3. 横竖屏适配优化：阅读页支持横竖屏切换，自动适配屏幕比例，图片自动调整适配；
4. 手机端省电优化：优化图片渲染逻辑，减少后台内存占用，避免长时间阅读导致手机发热。

## 3. 非功能需求
1. 性能要求：配置修改实时生效，无延迟；上下翻页模式滑动流畅，帧率≥60帧；全页面切换动画无卡顿；
2. 稳定性要求：连续阅读1小时无内存溢出、无页面崩溃；批量操作无数据错乱；配置永久保存，刷新页面不丢失；
3. 兼容性要求：全功能适配PC端主流浏览器、手机端安卓/iOS主流机型，无功能异常；
4. 易用性要求：所有操作符合用户直觉，无学习成本，核心功能3步内可触达。
