## 1. 项目背景
本项目为**纯个人自用**的本地漫画浏览网站，核心解决个人本地漫画文件的管理、浏览、个性化分类需求，无需依赖外部漫画源，无需适配多用户场景。
- 核心痛点：本地漫画文件分散、查找不便，通用阅读器无法满足个性化分类、评分、进度记忆等需求；
- 核心诉求：打造轻量、流畅、高度自定义的漫画浏览工具，同时适配PC端与手机端操作习惯；
- 存储约定：
  - 原始图片存储：`data/pictures/{漫画ID}/` 目录下，每个漫画对应一个文件夹，文件夹名称为漫画ID；
  - 元数据存储：`data/meta_data/comics_database.json` 文件，存放漫画ID、名称、标签、评分、阅读进度等所有元数据；
- 排除范围：无用户登录/权限管理、无反爬虫设计、无商用漫画源对接、无版权相关功能，所有资源均为个人本地合法持有。

## 2. 技术栈选型与说明
### 2.1 整体架构
前后端分离架构，后端提供RESTful API接口，前端负责页面渲染与交互，本地部署即可运行，无需云服务器。

### 2.2 详细技术栈
| 端侧 | 核心技术 | 配套依赖/工具 | 选型核心理由 |
|------|----------|---------------|--------------|
| 后端 | Python Flask 2.0+ | 1. 数据存储：JSON文件（无需数据库，轻量易维护）<br>2. 文件处理：Pillow（图片处理）、os/shutil（文件路径操作）<br>3. 接口适配：Flask-CORS（跨域处理）<br>4. 日志管理：logging（内置日志模块） | 轻量易上手，无冗余配置，第三方库生态完善，完全满足个人本地使用的性能需求，开发成本极低；JSON文件存储无需额外数据库服务，直接读写本地文件即可 |
| 前端 | Vue3 + Vite | 1. UI组件：Vant（移动端）、Element Plus（PC端可选）<br>2. 路由管理：Vue Router 4<br>3. 请求工具：Axios<br>4. 功能插件：vue-lazyload（图片懒加载）、pinia（状态管理） | 组件化开发易维护，Vant提供开箱即用的移动端响应式组件，Vue3生态成熟，学习成本低，适配跨端布局开发 |

## 3. 核心功能总览
### 3.1 页面整体架构
- 底部固定双页签导航：**主页**、**我的页面**，全端统一入口
- 核心页面流转路径：主页 → 漫画详情页 → 图片浏览页
- 全页面响应式适配：PC端/手机端自动适配布局，无样式错乱

### 3.2 核心功能模块
| 一级模块 | 二级功能点 | 核心能力说明 |
|----------|------------|--------------|
| 漫画资源管理 | 本地漫画导入 | 支持将漫画图片文件夹移动到 `data/pictures/{漫画ID}/`，漫画按照顺序命名为00001.webp、00002.webp等，名称为5位数。元数据存放在 `comics_database.json` 文件里。|
| | 漫画元数据管理 | 支持编辑漫画标题、作者、简介、封面，自动维护总页数、创建时间、最后阅读时间，所有变更同步到JSON文件 |
| 搜索与筛选 | 多维度搜索 | 支持漫画ID精准搜索、漫画名称模糊搜索、标签精准搜索，支持组合搜索 |
| | 多标签高级筛选 | 支持多标签「与」筛选（同时包含所有选中标签）、多标签「排除」筛选（不包含任意选中标签），支持两种模式组合使用 |
| 漫画详情管理 | 详情页展示 | 展示漫画封面、基础信息、标签、评分、章节缩略图、阅读进度 |
| | 评分系统 | 1-12分制评分，支持0.5分精度，评分实时保存与展示，支持按评分筛选 |
| 漫画阅读核心 | 无缝翻页 | 支持左右翻页、上下翻页两种模式，翻页无迟滞、图片无缝拼接，无视觉断层 |
| | 自定义懒加载 | 支持自定义预加载图片数量m，浏览第n张时自动预加载n+m张，不阻塞当前页面渲染 |
| | 图片操作 | 支持图片缩放（0.5倍-2倍）、重置缩放、适配屏幕宽度/高度，PC端滚轮缩放、手机端双指捏合缩放 |
| | 阅读进度管理 | 自动记忆当前阅读页码、翻页模式、缩放比例，退出后重进自动定位，实时展示「当前页/总页数」 |
| | 阅读体验优化 | 支持阅读背景切换（白色/深色/护眼色）、操作栏自动隐藏、快捷键操作、页码快速跳转 |
| 个性化管理 | 清单功能 | 支持创建自定义清单，漫画加入/移出清单，批量操作清单内漫画 |
| | 标签管理 | 支持自定义标签的新增/编辑/删除，漫画与标签多对多绑定，批量绑定标签 |
| | 系统配置 | 支持自定义预加载数量、默认翻页模式、默认背景色等配置，配置永久生效 |
| 跨端适配 | 响应式布局 | 全页面自动适配PC端/手机端屏幕比例，布局元素自动缩放与重排 |
| | 移动端操作 | 支持触屏滑动翻页、捏合缩放、侧滑返回、长按快捷操作，符合移动端用户习惯 |

## 4. 项目约束与边界
### 4.1 必须遵守的约束
1. 所有漫画图片必须存放在 `data/pictures/{漫画ID}/` 目录下，文件夹名称为漫画ID；
2. 所有元数据必须读写自 `data/meta_data/comics_database.json` 文件，格式需严格遵循约定；
3. 仅支持单用户本地使用，不开发多用户、权限管理、云端同步等功能；
4. 所有数据均存储在本地，无云端数据上传，保证数据隐私。

### 4.2 暂不实现的功能
- 用户注册/登录、角色权限管理
- 漫画在线爬虫、下载、源站对接
- 评论、弹幕、社交相关功能
- 多设备云端同步功能
- 付费、会员相关功能

## 6. 核心接口设计规范
### 6.1 统一接口风格
- 所有接口采用RESTful风格，请求方式遵循：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）
- 接口统一前缀：`/api/v1`
- 所有接口返回格式统一，如下：
```json
{
  "code": 200, // 状态码：200成功，4xx客户端错误，5xx服务端错误
  "msg": "操作成功", // 提示信息
  "data": {} // 业务数据，无数据时返回null
}
### 6.2 统一错误码定义
| 错误码 | 含义         | 适用场景                             |
| ------ | ------------ | ------------------------------------ |
| 200    | 成功         | 所有操作成功的场景                   |
| 400    | 参数错误     | 前端传参缺失、格式错误               |
| 404    | 资源不存在   | 漫画 ID 不存在、标签不存在、清单不存在 |
| 500    | 服务端错误   | 文件解析失败、数据库操作失败、图片读取失败 |
