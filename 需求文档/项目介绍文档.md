## 1. 项目背景
本项目为**纯个人自用**的本地漫画浏览网站，核心解决个人本地漫画文件的管理、浏览、个性化分类需求，无需依赖外部漫画源，无需适配多用户场景。
- 核心痛点：本地漫画文件分散、查找不便，通用阅读器无法满足个性化分类、评分、进度记忆等需求；
- 核心诉求：打造轻量、流畅、高度自定义的漫画浏览工具，同时适配PC端与手机端操作习惯；
- 存储约定：
  - 原始图片存储：`data/pictures/{漫画ID}/` 目录下，每个漫画对应一个文件夹，文件夹名称为漫画ID；
  - 元数据存储：`data/meta_data/comics_database.json` 文件，存放漫画ID、名称、标签、评分、阅读进度等所有元数据；
- 排除范围：无用户登录/权限管理、无反爬虫设计、无商用漫画源对接、无版权相关功能，所有资源均为个人本地合法持有。

## 2. 技术栈选型与说明
### 2.1 整体架构
前后端分离架构，后端提供RESTful API接口，前端负责页面渲染与交互，本地部署即可运行，无需云服务器。

### 2.2 详细技术栈
| 端侧 | 核心技术 | 配套依赖/工具 | 选型核心理由 |
|------|----------|---------------|--------------|
| 后端 | Python Flask 2.0+ | 1. 数据存储：JSON文件（无需数据库，轻量易维护）<br>2. 文件处理：Pillow（图片处理）、os/shutil（文件路径操作）<br>3. 接口适配：Flask-CORS（跨域处理）<br>4. 日志管理：logging（内置日志模块） | 轻量易上手，无冗余配置，第三方库生态完善，完全满足个人本地使用的性能需求，开发成本极低；JSON文件存储无需额外数据库服务，直接读写本地文件即可 |
| 前端 | Vue3 + Vite | 1. UI组件：Vant（移动端）、Element Plus（PC端可选）<br>2. 路由管理：Vue Router 4<br>3. 请求工具：Axios<br>4. 功能插件：vue-lazyload（图片懒加载）、pinia（状态管理） | 组件化开发易维护，Vant提供开箱即用的移动端响应式组件，Vue3生态成熟，学习成本低，适配跨端布局开发 |

## 4. 项目约束与边界
### 4.1 必须遵守的约束
1. 所有漫画图片必须存放在 `data/pictures/{漫画ID}/` 目录下，文件夹名称为漫画ID；
2. 所有元数据必须读写自 `data/meta_data/comics_database.json` 文件，格式需严格遵循约定；
3. 仅支持单用户本地使用，不开发多用户、权限管理、云端同步等功能；
4. 所有数据均存储在本地，无云端数据上传，保证数据隐私。

### 4.2 暂不实现的功能
- 用户注册/登录、角色权限管理
- 漫画在线爬虫、下载、源站对接
- 评论、弹幕、社交相关功能
- 多设备云端同步功能
- 付费、会员相关功能

## 6. 核心接口设计规范
### 6.1 统一接口风格
- 所有接口采用RESTful风格，请求方式遵循：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）
- 接口统一前缀：`/api/v1`
- 所有接口返回格式统一，如下：
```json
{
  "code": 200, // 状态码：200成功，4xx客户端错误，5xx服务端错误
  "msg": "操作成功", // 提示信息
  "data": {} // 业务数据，无数据时返回null
}
```
### 6.2 统一错误码定义
| 错误码 | 含义         | 适用场景                             |
| ------ | ------------ | ------------------------------------ |
| 200    | 成功         | 所有操作成功的场景                   |
| 400    | 参数错误     | 前端传参缺失、格式错误               |
| 404    | 资源不存在   | 漫画 ID 不存在、标签不存在、清单不存在 |
| 500    | 服务端错误   | 文件解析失败、数据库操作失败、图片读取失败 |

## 8. 图片处理规范
### 8.1 漫画图片存储
- 存储路径：`data/pictures/{漫画ID}/`
- 文件命名：5位数字序号，如 `00001.webp`、`00002.jpg`、`00003.png`
- 支持格式：jpg、png、webp
- 排序规则：按文件名自然排序（1, 2, 10 正确排序为 1, 2, 10）

### 8.2 封面缩略图规范
- 存储路径：`static/cover/{漫画ID}.jpg`
- 宽度：500px
- 高度：自适应（保持原始宽高比）
- 压缩质量：85%
- 格式：JPEG
- 生成时机：漫画初始化时自动生成

### 8.3 详情页缩略图
- 展示页码：第1、5、10、最后一页
- 处理方式：不压缩，直接使用原图
- 尺寸：根据容器自适应

## 9. 漫画导入流程
### 9.1 流程概述
漫画导入分为两个阶段：**数据准备阶段**和**系统初始化阶段**。

### 9.2 数据准备阶段（用户提供）
1. 准备漫画图片文件夹，命名格式：`{漫画ID}/`
2. 图片文件按阅读顺序命名：00001.jpg, 00002.jpg, ...
3. 运行用户提供的 Python 脚本，自动同步图片和元数据到 data 目录

### 9.3 系统初始化阶段（第一阶段开发）
1. 后端接收初始化请求
2. 校验图片目录是否存在
3. 解析图片文件，计算总页数
4. 生成封面缩略图
5. 写入元数据到 JSON 文件
6. 返回初始化结果

### 9.4 导入管理入口
- 位置：「我的页面」新增「导入漫画」按钮
- 功能：显示已导入漫画列表、手动触发重新初始化

## 10. 阅读模式技术方案
### 10.1 左右翻页模式
- 实现方式：传统单页切换
- 预加载策略：当前页 ± m 张图片预加载到浏览器缓存
- 翻页动画：CSS transition 实现平滑过渡

### 10.2 上下翻页模式（推荐方案B）
#### 10.2.1 CSS布局实现
```css
.reader-container-updown {
  display: flex;
  flex-direction: column;
  width: 100%;
  overflow-y: auto;
  scroll-behavior: smooth;
}

.comic-page {
  width: 100%;
  height: auto;
  display: block;
  margin: 0;
  padding: 0;
  border: none;
  object-fit: contain;
}
```

#### 10.2.2 虚拟滚动优化（长漫画必备）
- 仅渲染可视区域 + 前后3张图片
- 动态计算可视区域图片索引
- 避免DOM过多导致卡顿

#### 10.2.3 不同尺寸图片处理
- 宽度统一：100%，高度自适应
- 超高图：允许上下滑动查看
- 超宽图：object-fit: contain 保证完整显示
- 可选：服务端预处理统一裁剪到固定宽度

## 11. 性能优化方案
### 11.1 图片懒加载策略
- 预加载到浏览器缓存，不渲染到DOM
- 实现方式：使用 Image 对象预加载
```javascript
const preloadImages = (imageUrls) => {
  imageUrls.forEach(url => {
    const img = new Image();
    img.src = url;
  });
};
```

### 11.2 预加载参数
- 默认预加载数量 m=3
- 用户可配置范围：1-10
- 预加载时机：浏览第n张时，自动预加载n到n+m张

### 11.3 内存优化
- 虚拟滚动减少DOM节点
- 图片仅在可视区域渲染
- 及时释放不可见图片资源

## 13. 错误处理和边界情况
### 13.1 图片损坏或丢失
#### 后端处理
- 图片读取失败时，返回默认占位图
- 日志记录：`ERROR: 图片损坏 - comic_id: xxx, page: xxx, path: xxx`

#### 前端处理
- 图片 onerror 事件触发时，显示默认占位图
- 提示用户「第 X 页图片损坏」

### 13.2 JSON文件读写冲突
#### 解决方案：文件锁保证原子性
```python
import fcntl

def write_json_safe(data, filepath):
    with open(filepath, 'w', encoding='utf-8') as f:
        fcntl.flock(f.fileno(), fcntl.LOCK_EX)
        json.dump(data, f, ensure_ascii=False, indent=2)
        fcntl.flock(f.fileno(), fcntl.LOCK_UN)
```

### 13.3 JSON文件损坏
- 每次写入前自动备份为 `.bak` 文件
- 读取失败时尝试读取备份文件
- 备份文件也损坏时，创建新的空数据结构

### 13.4 其他边界情况
| 场景 | 处理方案 |
|------|----------|
| 漫画目录不存在 | 返回友好错误提示，不写入无效数据 |
| 目录无有效图片 | 返回提示，日志记录警告 |
| 请求参数缺失 | 返回400错误，说明缺少参数 |
| 访问不存在的漫画ID | 返回404，前端跳转回列表页 |
